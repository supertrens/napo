# Spirit of Haiti Air

> An airline for the diaspora, by the diaspora.

Single-page pledge platform for rallying the Haitian diaspora to commit toward a $25M raise to acquire a stake in Spirit Airlines. **Pledges only — no money moves through this site.** Real-time updates, premium UI, tier-based recognition, email-based deduplication.

---

## Stack

- **Next.js 16** (App Router, React 19, Turbopack)
- **Tailwind CSS v4** (theme tokens in `app/globals.css`)
- **Framer Motion** (entrance animations, live feed)
- **Convex** (database + reactive queries — no separate websocket layer needed)
- **Vercel** (deployment)

Why Convex instead of Firebase: reactive queries are first-class. Every component using `useQuery` automatically re-renders when underlying data changes. No subscription wiring, no manual websocket plumbing.

## One-time setup (≈ 3 minutes)

```bash
# 1. Install deps
npm install

# 2. Provision Convex — opens a browser to log in,
#    creates a project, writes NEXT_PUBLIC_CONVEX_URL to .env.local,
#    pushes the schema, generates convex/_generated/.
npx convex dev
```

Leave that terminal running. In another terminal:

```bash
# 3. Start Next.js
npm run dev
```

Visit http://localhost:3000.

> The first time you submit a pledge, you'll see it appear in the live feed within a few hundred milliseconds — Convex pushes the update to every connected client. Open two browser tabs to verify.

### Tier system

Tiers are **cumulative across all pledges from the same email**.

| Tier        | Sublabel   | Threshold |
| ----------- | ---------- | --------- |
| **Sipòtè**  | Supporter  | $50       |
| **Bilder**  | Builder    | $250      |
| **Patron**  | Patron     | $1,000    |
| **Pilye**   | Pillar     | $5,000    |
| **Fondatè** | Founder    | $10,000   |

When a pledger returns with the same email, the new amount **stacks** onto their existing total via a Convex mutation that detects the existing row by indexed lookup (`by_email`) and patches it. UI shows a "Welcome back, X — you've already pledged $Y" banner and prefills name/city/country from the prior submission.

### Real-time

- `useQuery(api.pledges.totals)` — animated counter on the hero
- `useQuery(api.pledges.recent, { limit: 12 })` — live feed
- `useQuery(api.pledges.recent, { limit: 1 })` — floating ticker

Convex pushes deltas via websocket. No polling, no SSE setup.

### Scale

The mutation maintains a singleton `stats` row (`totalAmount`, `totalPledgers`) updated atomically inside the same transaction as the pledge insert/update. The totals query reads one document — constant-time at any pledge count. The `recent` query uses an indexed range scan over `by_lastPledge`, also constant-time.

## Deploy to Vercel

This repo includes `vercel.json` that overrides the build command to:

```
npx convex deploy --cmd 'next build'
```

That deploys Convex schema + functions to production, then builds Next.js. **One env var is required for this to work.**

### Required: `CONVEX_DEPLOY_KEY`

Without this Vercel env var the build fails with:

```
✖ Error fetching GET https://api.convex.dev/.../team_and_project 401 Unauthorized:
  MissingAccessToken: An access token is required for this command.
  Authenticate with `npx convex dev`
```

**Steps:**

1. Open the [Convex dashboard](https://dashboard.convex.dev/).
2. Pick (or create) a **production** deployment for this project.
3. **Settings → URL & Deploy Key → Generate Production Deploy Key**. Copy it (starts with `prod:`).
4. Vercel project → **Settings → Environment Variables** → add:
   - Name: `CONVEX_DEPLOY_KEY`
   - Value: the key you copied
   - Environments: **Production** (and Preview if you want preview deploys)
5. Redeploy. `NEXT_PUBLIC_CONVEX_URL` is auto-injected by `convex deploy` during the build — no need to set it manually.

### One-time first deploy

1. Push this repo to GitHub.
2. Vercel dashboard → **Add New… → Project** → import the repo.
3. Add `CONVEX_DEPLOY_KEY` (above) before the first build.
4. Deploy. Subsequent `git push` to `main` triggers automatic deploys. If the Convex schema deploy fails, the Next.js build aborts — you won't ship a frontend pointing at an outdated backend.

## Data export

All pledge submissions live in your Convex project — full ownership, no third-party broker.

```bash
npx convex export --path ./pledges-export.zip
```

The `pledges` table contains: `name`, `email` (lowercased), `amount` (cumulative USD), `city`, `country`, `pledgeCount`, `lastPledgeAt`, plus Convex's `_id` and `_creationTime`.
